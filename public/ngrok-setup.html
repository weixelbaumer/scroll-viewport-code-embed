Ngrok Setup Guide

IMPORTANT: Ngrok warning messages can be bypassed by properly setting up your ngrok tunnel with the correct headers.

Correct Way to Start Ngrok

Use the following command to start ngrok with the proper header:

ngrok http 3000 --request-header-add="ngrok-skip-browser-warning: true"

Note the quotes and the space after the colon. This is critical.

Verifying Your Ngrok Tunnel

To test if your ngrok tunnel is properly set up, use this curl command:

curl -X GET https://YOUR-NGROK-URL.ngrok-free.app/atlassian-connect.json \
    -H "User-Agent: Mozilla/5.0" \
    -H "ngrok-skip-browser-warning: true"

You should see the JSON content of your Atlassian Connect descriptor.

Troubleshooting

Problem: Warning Still Appears

  • Make sure ngrok is running with the correct flags
  • Check that the server is properly setting the header on all responses
  • Try clearing your browser cache
  • Try using incognito/private browsing mode

Problem: "Your account is limited to 1 simultaneous ngrok agent sessions"

This means you already have an ngrok tunnel running. You need to:

  1. Find and terminate the existing ngrok process: 
    ps aux | grep ngrok
    kill -9 [PROCESS_ID]
  2. Start a new ngrok tunnel with the proper headers

Complete Setup (Step by Step)

  1. Kill any existing ngrok processes: 
    ps aux | grep ngrok
kill -9 [PROCESS_ID]
  2. Make sure your server is running: 
    cd /Users/weix/Code/Apryse/iText\ Confluence/scroll-viewport/scroll-viewport-code-embed
node server.js
  3. Start a new ngrok tunnel with proper headers: 
    ngrok http 3000 --request-header-add="ngrok-skip-browser-warning: true"
  4. Note the new ngrok URL and update your atlassian-connect.json file
  5. Test the connection: 
    curl -X GET https://YOUR-NGROK-URL.ngrok-free.app/atlassian-connect.json -H "User-Agent: Mozilla/5.0" -H "ngrok-skip-browser-warning: true"
